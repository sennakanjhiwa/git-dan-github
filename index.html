Repository dan Menambahkan File

https://git-scm.com/book/id/v2/Memulai-Pengaturan-Awal-Git

(git init) untuk mengubah folder menjadi repositori

Working Direktori, belum masuk ke repositori

(git status) untuk melakukan pengecekan folder/file, jika warna merah maka belum masuk ke staging/repositori/untracked(file baru yg belum pernah masuk ke repositori)/modified(file lama yg telah dimodifikasi) dan jika berwarna hijau maka masuk di stagging.

(git add .) untuk menambahkan semua file/folder ke staging

(git add "nama file") untuk menambahkan satu file ke staging

(git commit -m "Pesan yang perlu diisi") untuk memindahkan file dari staging ke repositori

Mengubah File dan Git Diff

(git diff) untuk mencari tahu apa yang berbeda dari yang sebelumnya/jika terjadi perubahan. Klik tombol "q" untuk keluar dari mode diff.

Melihat History dengan Git Log

(git log) untuk melihat histori/perubahan pada perjalanan development dan git kita. git log berisi:

commit 9f71fa3ac67f3ffcd499dd4f051c57696a39d808 (HEAD -> master)
Author: Senna Bhany Majid
Date: Mon Jul 3 21:40:41 2023 +0700

Memperbaiki tipo dan menambah penjelasan git diff

(git log --oneline) untuk mempersingkat git log. git log --oneline berisi:

9f71fa3 (HEAD -> master) Memperbaiki tipo dan menambah penjelasan git diff
cdb97e4 Mempelajari git init, git add, git status dan git commit

Membatalkan Perubahan dengan Git Restore

(git restore "nama file") untuk mengembalikan/membatalkan perubahan yang terjadi pada suatu file modified di working area

(git restore .) untuk mengembalikan/membatalkan perubahan pada semua file di working area

Git Reset dan Git Restore Staged

(git restore -S "nama file") untuk mengembalikan file modified dari staging area ke working direktori

(git reset .) untuk mengubah semua file di staging area ke working direktori

Kembali ke Commit Sebelumnya dengan Git Reset

(git reset --hard "nomor hash commit") untuk kembali ke commit sebelumnya sesuai dengan nomor hash yang digunakan dan menghapus commit setelahnya. Untuk melihat nomor hash lakukan git log --oneline

Branch dan Merge

(git checkout -b "nama branch") untuk membuat branch baru diluar branch master/main

(git checkout master) untuk berpindah ke branch master/main

(git merge "nama branch") untuk menggabungkan branch cabang ke branch master/main

Merevisi Commit dengan Git Revert

(git revert "nomor hash") untuk memperbaiki/merevisi commit sebelumnya, gunakan git log --oneline untuk mengcopy nomor hash commit yang akan direvisi. Tidak akan menghapus commit setelahnya, namun akan menimbulkan konflik.

Memulai dan Membuat Repository di Github

Setting SSH. Masuk ke halaman profile GitHub > Settings > SSH and GPG keys > generating SSH keys > Generate new SSH key (klik disini) > Add a new SSH key, Lalu copy SSHnya. Kembali ke sini, lalu klik New SSH key. Pastekan SSH key Laptop/PC Anda disana.

Membuat Repositori Baru

echo "# git-dan-github" >> README.md

git init

git add README.md

git commit -m "first commit"

git branch -M main

git remote add origin git@github.com:kanjhiwa/git-dan-github.git

git push -u origin main

Git Clone dan Git Push

Masuk ke repositori, lalu pilih code dan clone menggunakan SSH. Contoh sebagai berikut.
(git clone git@github.com:kanjhiwa/git-dan-github.git) untuk mengclone dari repositori tersebut.

(git push origin main) untuk melakukan push atau update perubahan terbaru ke GitHub. Gunakan git push origin master jika sebelumnya tidak diubah ke main.

Github Pull Request

(git checkout -b "nama branch") untuk membuat branch baru atau branch cabang

(git push origin "nama branch cabang") untuk melakukan pull request di GitHub atau push ke GitHub dengan branch cabang yang nantinya masuk ke Pull Request dan kita create pull request. Jika oleh maintener code yang telah kita create pull request sudah sesuai keinginan, maka maintener akan melakukan merge pull request sehingga code kita akan digabungkan ke branch main/master.

(git pull origin main) untuk menyingkronkan code kita dengan yang ada di GitHub. Hal ini penting agar kita mendapatkan perubahan terbaru yang terjadi di repositori GitHub yang sedang kita remote.

Aturan Menulis Pesan Saat Commit

Pesan pada saat commit sangatlah penting, oleh karena itu kita perlu menuliskan pesan dengan baik dan benar. Kunjungi situs https://www.conventionalcommits.org/en/v1.0.0/ untuk melihat penulisan commit yang rapi dan baik.
